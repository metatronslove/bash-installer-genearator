# Bash Installer Generator (BIG)

Bash Installer Generator (BIG) is a terminal-based tool that creates self-contained bash scripts for installing files on Linux systems. It embeds files in base64 format with zlib compression, allowing distribution and installation without external dependencies like `.deb`, `.rpm`, or `.tar.gz` files. Files are added recursively from a directory, with a predefined `post-install-instructions.sh` used for post-installation tasks, and the TUI provides a clean, dynamic interface.

## Features
- **Terminal User Interface (TUI)**: Built with `prompt_toolkit` for an interactive console experience with path autocompletion.
- **Recursive Directory Support**: Add files from a directory and its subdirectories, selecting from a tree-like display.
- **Predefined Post-Installation Script**: Uses `post-install-instructions.sh` from the directory root, excluded from embedded files.
- **Dynamic Menu**: Menu items show current state (e.g., number of files, output file name) in parentheses.
- **Clean Console**: Console clears before each menu display and on exit to avoid clutter and restore the previous state.
- **File Embedding**: Compresses files with `zlib` and encodes them in base64 within a JSON structure.
- **Customizable Paths**: Specify target installation paths for each file.
- **Colored Output**: Generated scripts use ANSI colors for progress (yellow), success (green), and errors (red).
- **Self-Installation**: Generate a script to install BIG itself, including dependencies.

## Prerequisites
- **Python 3**: Required to run the TUI.
- **prompt_toolkit**: Python library for the TUI (installed via `requirements.txt`).
- **Linux Environment**: Generated scripts require `base64`, `zcat`, and `mkdir` (standard on most Linux distributions).
- **pip3**: Required for installing dependencies.
- **tree (optional)**: Enhances directory tree display (`sudo apt install tree` on Debian/Ubuntu or `sudo pacman -S tree` on Arch).

## Installation
To install BIG, use the self-installing script generated by the TUI.

### Step 1: Set Up the TUI
1. Clone the repository:
   ```bash
   git clone https://github.com/username/big.git
   cd big
   ```
2. Install dependencies from `requirements.txt`:
   ```bash
   pip install -r requirements.txt --user
   ```
3. (Optional) Install `tree` for enhanced directory display:
   ```bash
   sudo apt install tree  # Debian/Ubuntu
   sudo pacman -S tree   # Arch Linux
   ```
4. Create `post-install-instructions.sh` in the directory root (e.g., `big/`):
   ```bash
   nano post-install-instructions.sh
   ```
   Add:
   ```bash
   # Check for python3
   command -v python3 >/dev/null 2>&1 || { echo -e "\033[0;31mError: python3 is required but not found\033[0m"; exit 1; }
   # Check for pip
   command -v pip3 >/dev/null 2>&1 || { echo -e "\033[0;31mError: pip3 is required but not found\033[0m"; exit 1; }
   # Install prompt_toolkit
   pip3 install prompt_toolkit --user >/dev/null 2>&1
   if [ $? -eq 0 ]; then
       echo -e "\033[0;32mSuccessfully installed prompt_toolkit\033[0m"
   else
       echo -e "\033[0;31mFailed to install prompt_toolkit\033[0m"
       exit 1
   fi
   # Ensure big is executable
   chmod +x /usr/local/bin/big 2>/dev/null
   ```
   Save and exit.
5. Make the TUI executable:
   ```bash
   chmod +x installer_tui.py
   ```

### Step 2: Generate the Self-Installer
1. Run the TUI:
   ```bash
   ./installer_tui.py
   ```
2. Follow these steps in the TUI:
   - Select `1` to add a directory recursively (e.g., enter `./` for the current directory).
   - View the directory tree (uses `tree` if available, else a numbered list).
   - Select files by entering indices (e.g., `1` for `installer_tui.py`) or `all` for all files except `post-install-instructions.sh`.
   - The TUI automatically uses `post-install-instructions.sh` as the post-installation script if present.
   - Select `2` to set the target path for `installer_tui.py` to `/usr/local/bin/big`.
   - Select `3` to set the output name to `install_big.sh`.
   - Select `4` to generate the script.
3. The TUI clears the console before each menu display and on exit.

### Step 3: Run the Self-Installer
1. Make the script executable:
   ```bash
   chmod +x install_big.sh
   ```
2. Run it (requires `sudo` for `/usr/local/bin`):
   ```bash
   sudo ./install_big.sh
   ```
3. Verify installation:
   ```bash
   big
   ```

## Usage
1. Run the TUI:
   ```bash
   big
   ```
2. Use the menu to:
   - **Add directory (recursive)** (`1`): Add files from a directory and subdirectories, selecting from a tree-like display. Uses `post-install-instructions.sh` if present.
   - **Update target paths** (`2`): Specify installation paths (shows file count).
   - **Set output script name** (`3`): Name the generated script (shows current name).
   - **Generate bash script** (`4`): Create the installer.
   - **Exit** (`5`): Exit and clear the console.
3. The console clears before each menu display to avoid clutter.
4. Run the generated script:
   ```bash
   chmod +x installer.sh
   ./installer.sh
   ```
   Note: `sudo` may be required for system paths.

## Example Generated Script
Generated scripts:
- Embed files (except `post-install-instructions.sh`) in a JSON structure with base64-encoded, zlib-compressed content.
- Check for required commands (`base64`, `mkdir`).
- Install files to specified paths, creating directories as needed.
- Execute the post-installation script from `post-install-instructions.sh` if provided.
- Use ANSI colors for progress (yellow), success (green), and errors (red).
